Industrial HMI · Neocoude HD-15

Industrial Machine Retrofit: Neocoude HD-15 Rebar Bender — Web HMI via Modbus RTU

Full modernization of a 15 HP industrial hydraulic rebar bender (Neocoude HD-15, capacity up to ⌀50 mm rebar). The original HMI panel was damaged — hardware replacement would have cost €2,000+. Instead, I bridged the gap between raw hardware and user-friendly software by building a modern Web HMI running on a Raspberry Pi 3B+, communicating directly with the machine's PLC via Modbus RTU over RS485. Any tablet or smartphone on the local network becomes the operator interface.

Business Impact

Eliminated a €2,000+ hardware replacement cost while adding capabilities the original panel never had: remote monitoring, automatic 7-step calibration, production data logging, and Tailscale VPN for off-site access. This is the ROI that makes retrofit engagements compelling: spend less, get significantly more.

Latency: <50ms
Poll Rate: 4 Hz
Precision: ±0.9°

Technical Implementation

  • Protocol: Modbus RTU @ 57,600 baud via RS485 (USB-RS485 converter)
  • Backend: Python asyncio server + WebSocket for real-time UI push
  • Encoder: 400 PPR incremental encoder with 32-bit position tracking
  • Calibration: 7-step automatic calibration (sensor mapping + ballistic inertia compensation)
  • State Machine: 8-state control logic (IDLE, BENDING, RETURNING, CALIBRATION…)
  • Remote Access: WiFi dual-mode (AP + STA) + Tailscale VPN

Distributed KVM System

Distributed Hardware KVM System — Raspberry Pi + HID Emulation

Custom Keyboard, Video, and Mouse (KVM) emulation architecture for remotely controlling multiple physical machines over the network. Developed a scalable distributed system that routes keyboard and mouse inputs from a central Raspberry Pi server to multiple target computers via Raspberry Pi Pico W2 client nodes — each appearing as a native USB HID device, invisible to the host OS, requiring zero drivers.

System Architecture

  • Server Node: Raspberry Pi 4 — central control hub, input capture and routing, web UI
  • Client Nodes: Raspberry Pi Pico W2 — C/C++ firmware for USB HID emulation
  • Protocol: Custom low-latency network protocol for real-time input forwarding over Wi-Fi
  • HID Emulation: Pico W2 presents as native keyboard + mouse — plug-and-play on any OS
  • Scalability: Easily extended to N target machines on the same network segment
  • Use Cases: Industrial lab management, server rooms, multi-machine test benches

Business Impact

Replaces commercial KVM switches ($200–$1,000+) with a fully custom, extensible solution. Ideal for industrial labs and manufacturing environments where managing multiple embedded computers is routine and physical access is impractical.
